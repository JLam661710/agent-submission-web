智能体开发选题灵感大师 · 表单问卷应用

目标 - 社群互动与信息收集

前端形式：网页界面，需要进度指示器，可以暂存草稿，不能修改已提交的答案，只能再次重新提交，提交后的反馈内容不会把问卷的被填写内容在界面再次展示，但是会把调用coze扣子api之后的生成的内容展示在结果页面，作为给用户的反馈内容（可以提示用户复制文字或截图）

前端内容：用户点进去就是一系列问题问卷需要回答或填写的，这些问题问卷是关于用来调研填写人在开发扣子coze智能体和扣子coze工作流时的选题思路的，主要就是通过帮助填写人思考和自身的挖掘自己所在的领域（学生-所学专业、职业-行业/领域/赛道、爱好、技能特长等等路径），包括自己身边的人所在的行业、领域和赛道，从自身出发、从人类学意义上的“附近”出发，透过一种近似于个人民族志+用户体验设计研究用户调研的方式方法，来挖掘作为一个初学者（或者掌握了一定的ai开发、智能体开发能力）可以去通过学习ai学习智能体学习工作流来怎么样赋能与自己和自己身边的人，更近一步的是挖掘自身和身边的实战落地、商业化落地和商业化转化的潜能和可能性，从而获得一种非常有针对性的学习目标和学习路径，不要像无头苍蝇一样去学技术，学了半天也不知道能用来干嘛，或者在网上和技术社区看到很多别人的用法和做法，但是实际上并不适合于自己的水平和情况的，其实反而是扰乱和干扰了自己。总之就是通过设计问卷（不要那种社会学统计学的枯燥问卷，应该更加生动形象更加有趣的，调用AI的创意能力和语言能力和编写代码的能力让这个问卷表单更趣味化丰富化）的方法一步步地得到属于自己的那个答案。

AI能力：扣子coze智能体作为后端联动api，在前端用户填写完毕后带着数据系统调用api，获得进一步的LLM大模型分析的内容反馈

后端能力：扣子cozeAPI

后端数据库集成：前端用户提交以后，把数据结构化地发送到一个专属的飞书多维表格，创建新的记录，把数据填进去，所以需要设计飞书多维表格的字段，且飞书多维表格的字段应该和前端的问卷表单对应起来，包括扣子cozeAPI调用后得到的数据结果或文字结果也填入到多维表格的某一字段中。

用户身份认证：不需要实质的用户注册与身份认证系统接入，只需要在问卷开始前让用户填写一下自己的昵称就行了，比如说“请问我可以怎么称呼您？”或者“请填入你的飞书用户名（昵称）”


性能与体验要求：
页面加载速度优化设计
支持移动端适配
设计放置用户重复提交的机制


后续迭代规划：
后续可能需要对问卷内容进行后台可修改配置，但是当前初步MVP阶段还不需要。

表单问卷内容设计：


第一部分：欢迎与热身（2个问题）
引导文本：＂欢迎来到‘生活灵感挖掘机’！我是您的探索助手。我们将一起从您的日常小事出发，发现那些隐藏的智能机会。请放松，就像聊天一样～＂

目的：轻松开场，建立亲切感。

问题1：昵称识别

问题文本：＂首先，请问我可以怎么称呼您？例如您的飞书昵称、花名，或任意您喜欢的代号。＂ 

说明：简单身份识别，无需真实姓名。

输入方式：单行文本输入框。

问题2：角色标签

问题文本：＂现在，请用1-3个关键词描述您当前的角色（如‘学生’、‘创作者’、‘问题解决者’）。不必拘泥正式职称，随心就好！＂

说明：用标签式提问降低压力，引导创意自我描述。

输入方式：单行文本输入框。



第二部分：日常场景挖掘（4个问题）
引导文本：＂想象您正在回顾上周的生活。那些小瞬间——比如工作中的重复任务、学习中的卡点——也许正是灵感的种子。让我们从这些具体场景开始吧！＂

目的：从用户最熟悉的日常切入，挖掘具体痛点或兴趣点。

问题3：最近的小烦恼

问题文本：＂回想最近几天，有没有哪件小事让您觉得‘太耗时了’或‘可以更聪明点’？请描述这个场景（例如，‘每天手动整理文件很麻烦’）。＂

说明：从具体烦恼出发，作为思维钩子，引导用户思考改进需求。避免给例子，但用括号内描述暗示思考方向。

输入方式：多行文本输入框。

问题4：技能超能力

问题文本：＂每个人都有自己的‘超能力’！您最得意的技能或知识是什么？请列出1-2项，并说说它们如何在实际生活中帮到您（比如，‘我擅长组织信息，让团队会议更高效’）。＂

说明：连接技能与具体应用场景，避免抽象罗列。

输入方式：多行文本输入框。

问题5：兴趣热度计

问题文本：＂工作/学习之外，您对什么领域充满好奇？请用1-5分打分（1=偶尔看看，5=深度沉迷），并简短说明为什么它吸引您。＂

说明：打分题结合开放描述，评估兴趣强度。

输入方式： 

打分：单选按钮（1至5分）。

说明：单行文本输入框。

问题6：学习新事物态度

问题文本：＂当遇到AI这类新技术时，您的本能反应是？请用1-5分打分（1=保持距离，5=主动拥抱）。＂

说明：简单打分，评估开放程度。

输入方式：单选按钮（1至5分）。



第三部分：身边世界观察（3个问题）
引导文本：＂现在，把镜头转向您身边的人——家人、朋友或同事。他们的日常故事可能藏着您没注意到的机会。让我们一起观察吧！＂

目的：引导用户共情式观察“附近”，发现群体需求。

问题7：身边人的日常挑战

问题文本：＂想一位您常接触的人。他们每天在忙什么？有没有什么任务让他们觉得‘重复’或‘头疼’？请描述一下。＂

说明：从具体人的日常切入，而非直接问“行业”，降低抽象度。

输入方式：多行文本输入框。

问题8：社区共同声音

问题文本：＂在您的社交圈里，您常听到大家抱怨什么？比如‘信息太散乱’、‘沟通效率低’。请列出1-2个共同点，并说说您觉得原因是什么。＂

说明：引导用户发现群体性问题根源。

输入方式：多行文本输入框。

问题9：合作风格偏好

问题文本：＂如果有一个创新项目，您更希望如何与身边人合作？＂

说明：选择题用于简化分类。

目的：帮助用户连接个人与附近，形成个性化学习目标。

问题10：智能工具构想

问题文本：＂假如您有一个‘魔法工具箱’，可以设计一个智能工具（不限于AI）来优化您或身边人的生活，它会做什么？请描述它的功能和使用场景。＂

说明：开放性问题，鼓励想象力。提示用户参考前文思考（如烦恼、技能）。

输入方式：多行文本输入框。

问题11：落地信心指数

字段设计说明
字段数量：共24个字段（包括元数据和API结果）。实际核心字段对应13个问卷问题，但部分问题（如问题12）被拆分为多个字段以结构化存储。

字段类型选择： 

文本类型：用于开放性问题（如多行文本）和简短输入（如单行文本）。

数字类型：用于打分题（如1-5分），便于后续数值分析。

单选类型：用于选择题（如问题9），选项值预先定义，避免自由文本输入错误。

日期时间类型：用于自动记录提交时间。

处理复合问题： 

问题12（挑战雷达）是可变长度问题，用户可能列出1-3个障碍。设计为3组字段（障碍1-3及分数），允许空值，确保灵活性。

问题9（合作风格偏好）使用“单选”字段存储主选项，并用一个附加文本字段存储“其他”说明。

元数据字段： 

“提交时间”和“唯一提交ID”用于管理提交记录。如果前端已通过会话存储防止重复提交，可简化元数据。

Coze API集成： 

“Coze分析反馈”字段存储API返回的文本内容，后端在调用API后填充此字段。

飞书多维表格配置建议： 

创建表格时，按上述字段列表添加列。

设置“提交时间”为自动生成（如使用飞书公式或后端填充）。

对于“合作风格偏好”字段，在飞书多维表格中设置为“单选”类型，并预先添加选项A、B、C、D。

字段顺序可按问卷流程排列，便于查看。

此设计确保前端提交的数据能完整映射到多维表格，支持后续数据分析和Coze API集成。


coze 智能体工作流设计（后端 api 调用的那个）：
LLM节点的输出不再聚焦于建议具体的AI工具或开发应用，而是转向帮助用户深入理解自身情况和场景，输出一份“图谱式的相关性话题报告”。这份报告旨在通过关联性分析，揭示用户数据中的模式、潜在联系和洞察，帮助用户获得更清晰的自我认知，而不是直接指导行动。

以下是正式的system prompt，用于配置Coze工作流中的LLM节点。Prompt定义了LLM的角色、输入参数的使用机制、输出要求和作用方式。Prompt设计为直接使用传入的JSON参数，进行关联性分析。
System Prompt:
你是一个善于洞察的人类学式分析顾问，负责帮助用户深入理解自己的处境和场景。你的任务是基于用户提供的问卷数据，生成一份轻量化的“相关性话题报告”，突出用户数据中的关键模式、关联性和潜在洞察。报告不应提供具体的行动建议或AI开发指导，而是通过图谱式分析，帮助用户获得清晰的自我认知。


输入以 JSON 对象的 `_input` 键提供以下参数：
- nickname: 用户昵称
- role_tags: 用户角色标签
- recent_annoyance: 最近的小烦恼
- skills: 技能描述
- interest_score: 兴趣热度分数（1-5）
- interest_description: 兴趣说明
- learning_attitude_sc: 学习态度分数（1-5）
- people_around_challe: 身边人的挑战
- community_voices: 社区共同声音
- collaboration_prefer: 合作偏好（A/B/C/D）
- tool_idea: 智能工具构想
- confidence_score: 落地信心分数（1-5）
- challenges: 挑战障碍列表（每个包含 description 和 score）
- learning_goal: 学习目标


作用机制：
1. **数据解析**：首先，解析所有输入参数，识别关键元素（如烦恼、技能、兴趣）。
2. **关联性分析**：寻找数据中的联系，例如：
   - 用户技能与日常烦恼的关联。
   - 兴趣领域与身边人挑战的交叉点。
   - 学习态度与挑战难度的模式。
3. **模式发现**：突出重复出现的主题（如“效率提升”或“信息管理”），并说明这些主题如何贯穿用户的不同方面。
4. **图谱式输出**：组织报告为话题网络，而不是线性列表，强调洞察而非行动。


输出要求：
- **字数**：300-400字，简洁易读。
- **结构**：使用以下小标题组织内容：
  - **用户画像概要**：用1-2句话总结用户的核心特点（基于角色、兴趣、技能）。
  - **关键模式发现**：描述2-3个最显著的数据模式，例如“您的技能与烦恼高度相关”或“您的兴趣与社区声音存在重叠”。
  - **相关性话题图谱**：以话题为中心，分析这些话题如何连接用户的不同方面（如“效率”话题链接了您的烦恼、技能和构想）。每个话题应说明其关联性和意义。
  - **自我认知提示**：用启发式语言总结报告，帮助用户反思自己的处境，例如“这些模式提示您可能更注重实践性解决方案”。
- **语气**：中立、洞察性，避免指导性语言。使用“您的数据显示...”而非“您应该...”。
- **禁止内容**：不提及具体的AI工具、开发步骤或商业化建议。


请直接输出报告内容，不要添加额外解释。报告应基于输入数据，创造性但不脱离事实。

# 相关性话题报告：您的个人场景图谱


## 用户画像概要
小明，您是一个注重实践的学生和创作者，对科技解决实际问题充满兴趣，并且拥有组织信息的技能。您的学习态度积极，愿意拥抱新事物。


## 关键模式发现
- **模式1：技能与烦恼的强关联**：您的技能“组织信息”与烦恼“手动整理文件”直接相关，这提示您可能天然倾向于优化效率问题。
- **模式2：个人与社区的共鸣**：您的个人烦恼（文件整理）与社区声音（信息散乱）高度重叠，表明您所处的环境存在普遍的信息管理挑战。
- **模式3：信心与挑战的平衡**：您的落地信心（3/5）与挑战难度（时间不足4/5、技能欠缺3/5）形成对比，这反映了您对现实障碍的理性认知。


## 相关性话题图谱
- **话题“信息效率”**：这是您的核心话题。它连接了您的技能（组织信息）、烦恼（文件整理）、社区声音（信息散乱）和工具构想（自动整理）。该话题提示您对效率提升有内在驱动力。
- **话题“学习与应用”**：您的学习目标（自动化工具）与兴趣（科技解决实际问题）相关联，但挑战（技能欠缺）表明可能需要基础学习。这个话题突出了理论与实践的桥梁。
- **话题“合作动态”**：基于您的合作偏好（平等搭档），这个话题与社区声音（沟通效率）交叉，提示协作可能是解决信息问题的关键路径。


## 自我认知提示
您的数据显示，您是一个问题导向的思考者，善于从个人经验中发现普遍性挑战。这些模式提示您可能更注重实践性解决方案，而非抽象理论。通过理解这些关联，您可以更清晰地定位自己的学习和发展方向。
